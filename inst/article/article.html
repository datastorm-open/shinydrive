shinydrive

Dans la famille des packages shiny chez Datastorm, je demande le petit dernier, savoir shinydrive.

Le but de shinydrive est simple : faciliter le partage de fichiers entre diffrents utilisateurs, le tout avec un module insrer en quelques lignes de code dans une application R/shiny existante. Le module disposant de deux rles :

  • ladministrateur qui peut de son ct ajouter / modifier / supprimer des fichiers disponibles ensuite en tlchargement, et crer si besoin galement des sous-dossiers

  • le lecteur qui peut uniquement tlcharger un ou plusieurs fichiers disponibles

Le fonctionnement du package est trs simple :

  • Le module se branche un rpertoire existant
  • La gestion des fichiers imports se fait ensuite via un fichier yaml permettant de stocker :
    • le nom dorigine du fichier ainsi que sa date et heure de mise disposition. Cela permet en effet de renommer les fichiers dont le rpertoire de stockage en vitant les conflits dans le cas ou ladministrateur ajouterai deux fichiers portant le mme nom. Cela est totalement transparent pour le lecteur qui retrouvera bien le fichier avec le nom dorigine lors du tlchargement
    • le type de fichier
    • une description optionnelle de son contenu

Utilisation

En insrant tout simplement le module dans votre application shiny, avec lutilisation des deux fonctions suivantes :

  • shiny_drive_ui : dans le script ui.R, avec uniquement lidentifiant du module renseigner
  • shiny_drive_server : dans la partie du server.R avec minima :
    • lidentifiant du module (id)
    • ainsi que le rpertoire de stockage (save_dir)

En complment, il est possible de :

  • dfinir le rle, administrateur ou lecteur (admin_user)
  • la langue (lan)
  • de forcer le remplissage du champ description lors de linsertion dun nouveau fichier (force_desc)
  • et de restreindre les accs un sous-ensemble de sous-dossiers si ncessaire (dir_access)
require(shinydrive)

ui <- fluidPage(
    shiny_drive_ui(id = "idm")
)

server <- function(input, output, session) {
    callModule(module = shiny_drive_server,
             id = "idm",
             admin_user = TRUE,
             save_dir =  getwd(),
             lan = "EN")
}

shinyApp(ui, server)

Finalement, il est galement possible dutiliser de grer les fichiers en dehors de lapplication shiny (gnration quotidienne dun rapport via un batch par exemple, et mise disposition automatique dans une application) en utilisant les trois fonctions suivantes :

  • add_file_in_dir pour ajouter un nouveau fichier
  • edit_file_in_dir pour diter un fichier existant
  • suppress_file_in_dir pour supprimer un fichier
